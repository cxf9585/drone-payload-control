# 无人机载荷控制模块设计

## 目标

在 STM32 侧提供一个独立的载荷控制模块，用于接收飞控或任务计算机命令，并统一调度云台、相机、夹爪、喷洒等载荷动作。

当前代码位置：

- `STM32/PayloadControl/include/payload_control.h`：核心控制接口。
- `STM32/PayloadControl/src/payload_control.c`：状态机和安全检查。
- `STM32/PayloadControl/include/payload_mavlink_bridge.h`：MAVLink 命令桥接接口。
- `STM32/PayloadControl/src/payload_mavlink_bridge.c`：常用 MAVLink `COMMAND_LONG` 映射。
- `STM32/PayloadControl/examples/stm32_payload_app.c`：STM32 硬件适配示例。
- `STM32/PayloadControl/tests/test_payload_control.c`：主机侧单元测试。

## 模块边界

核心控制层不直接访问 GPIO、PWM、UART、CAN 或 I2C。所有硬件动作都通过 `payload_driver_t` 函数表注入。

这样可以保持：

- 飞控命令解析和业务状态机可测试。
- 具体硬件驱动可替换。
- 安全逻辑集中，避免散落在多个外设回调里。

## 状态机

- `PAYLOAD_STATE_DISARMED`：默认状态，拒绝实际载荷动作。
- `PAYLOAD_STATE_IDLE`：已解锁，等待命令。
- `PAYLOAD_STATE_BUSY`：命令执行中。
- `PAYLOAD_STATE_ERROR`：硬件动作失败或安全动作失败。

载荷动作必须先执行 `PAYLOAD_CMD_ARM`。急停 `PAYLOAD_CMD_ESTOP` 会调用 `all_outputs_safe`，并强制回到未解锁状态。

## 支持命令

- 解锁和上锁：`PAYLOAD_CMD_ARM`、`PAYLOAD_CMD_DISARM`
- 模式切换：`PAYLOAD_CMD_SET_MODE`
- 云台角度：`PAYLOAD_CMD_GIMBAL_SET_ANGLES`
- 拍照和录像：`PAYLOAD_CMD_CAMERA_TRIGGER`、`PAYLOAD_CMD_CAMERA_RECORD_START`、`PAYLOAD_CMD_CAMERA_RECORD_STOP`
- 夹爪：`PAYLOAD_CMD_GRIPPER_OPEN`、`PAYLOAD_CMD_GRIPPER_CLOSE`
- 喷洒：`PAYLOAD_CMD_SPRAYER_SET_RATE`
- 急停：`PAYLOAD_CMD_ESTOP`

## 待补硬件参数

请补齐以下信息后，可以把 `examples/stm32_payload_app.c` 中的 TODO 换成真实驱动：

| 项目 | 当前待定内容 |
| --- | --- |
| STM32 型号 | 例如 STM32F405、STM32H743、STM32G431 |
| 飞控链路 | UART MAVLink、CAN、PX4 内部接口或其他协议 |
| 载荷类型 | 云台、相机、夹爪、喷洒、抛投、绞盘或自定义载荷 |
| 外设接口 | GPIO、PWM、UART、CAN、RS485、I2C、SPI 或以太网 |
| 供电控制 | 是否有独立电源使能、继电器或电子开关 |
| 安全动作 | 急停时断电、停止 PWM、关闭泵、释放或保持 |
| 行程限制 | 舵机 PWM 范围、云台角度范围、泵最大流量 |
| 反馈信号 | 限位开关、电流检测、编码器、相机状态回传 |

## 建议接入顺序

1. 先实现 `all_outputs_safe`，确保急停和上锁可靠。
2. 再接入单个载荷动作，例如相机触发或夹爪开合。
3. 加入限位和反馈检测。
4. 接入 MAVLink ACK 或飞控遥测上报。
5. 最后做整机联调，以及失联、低电压、任务中断测试。

## 验证

本目录已提供 CMake 和 C 单元测试。当前机器没有安装 `gcc`、`clang`、MSVC `cl` 或 `cmake`，所以尚未在本机完成编译运行。

装好工具链后，在 `STM32/PayloadControl` 执行：

```sh
cmake -S . -B build
cmake --build build
ctest --test-dir build
```
